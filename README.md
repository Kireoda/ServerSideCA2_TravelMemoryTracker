CA2 Server-Side Development
MVC Laravel Application
Project Overview
This project is a core component of the Server-Side Development module. Students are required to plan, design, and develop a dynamic, data-driven web application using the Laravel framework and a Model–View–Controller (MVC) architecture.

Students will define their own project concept, scope, and user requirements, reflecting real-world software development practice. The project places strong emphasis on project planning, user experience design, collaborative development, and the ability to explain and justify technical decisions.

The use of AI-assisted development tools (such as VS Code agents or similar) is permitted. However, students remain fully responsible for the correctness, security, and quality of their application, and must be able to clearly explain how their system works during an individual oral assessment. Projects may be completed in pairs or small groups. Where pair work is used, each student will be assessed individually.
Deliverables
Students must submit the following:

Project Planning and UX Artefacts in PDF format, including:
Project concept and problem statement
Two clearly defined user personas, outlining goals, needs, and behaviours
User flows or task journeys based on these personas
Low-fidelity wireframes demonstrating key interactions
A clearly defined project scope, including features included and excluded

Working Laravel Application, which must:
Use a Model–View–Controller (MVC) architecture
Include at least one fully implemented CRUD feature (Create, Read, Update, Delete)
Store and manage data using a relational database and Laravel migrations
Implement user authentication and appropriate authorisation
Apply both client-side and server-side validation
Contain a minimum of five appropriate pages or views
Projects with a broader scope, additional CRUD features, or more advanced functionality may achieve higher grades, provided all features are well planned, implemented, and justified.

Git Repository, demonstrating:
Regular, meaningful commits throughout development
Use of branches and merges where appropriate
Clear commit messages that describe development decisions and changes

Project Documentation (README.md), outlining:
Project overview and purpose
Setup and installation instructions
Summary of implemented features
Any assumptions, limitations, or known issues

Individual Oral Project Defence (interview), in which each student:
Explains project planning and scope decisions
Discusses UX design choices, including the two user personas
Walks through the CRUD functionality and supporting code
Reviews aspects of the Git history and collaboration
Reflects on challenges, problem-solving, and use of AI-assisted development tools
Assessment Criteria
Your project will be assessed based on the following:

Project Planning and Scope Definition (20%)
The clarity and quality of the project concept, including how effectively the problem is defined, the realism and coherence of the project scope, evidence of feature prioritisation, and alignment between initial planning and the final implemented application.

User Experience Design and User-Centred Thinking (20%)
The effectiveness of UX preparation, including the quality and relevance of the two user personas, the use of user flows and wireframes to inform design decisions, consideration of usability and validation, and consistency between planned user experience and the implemented interface.

Technical Implementation (25%)
The successful development of a functional Laravel application, demonstrating appropriate use of MVC architecture, correct implementation of at least one CRUD feature, effective handling of dynamic data, authentication, and validation, and overall application stability.

Version Control and Collaboration (15%)
The extent to which Git is used effectively to support development, including meaningful commit history, appropriate use of branches and merges, clear evidence of individual contribution in pair work, and a logical progression of the project over time.

Oral Project Defence (20%)
The student’s ability to clearly and confidently explain and justify project planning, UX decisions, technical implementation, and problem-solving approaches during the oral assessment, using appropriate technical terminology and responding accurately to follow-up questions.
